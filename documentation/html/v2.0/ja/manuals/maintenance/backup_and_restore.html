2. データバックアップ・リストア¶

2.1. 目的¶

本頁では、Exastroシステムで利用する永続データのバックアップとリストア手順について説明します。
パブリッククラウドの仕組みにより、データバックアップの仕組みが利用可能であればそちらを利用しても問題ありません。

2.2. 前提条件¶

本頁で紹介するバックアップ・リストア手順は下記の条件を満たしている必要があります。

  • 条件

    • 下記のコマンドが利用可能なこと

      • tar

      • kubectl

    • 作業環境のサーバで充分なディスクの空き容量があること

2.3. 概要¶

バックアップ対象となるデータは下記の3つです。

  • バックアップ対象

    • Exastro Platform のデータベース

    • Exastro IT Automation のデータベース

    • Exastro IT Automation の共有ファイル

これらのデータを kubectl コマンドを利用してインストールします。
作業の流れは、まず、ユーザからのデータの書き込みを制限するために、リバースプロキシを停止します。
次に、バックヤード処理を停止したら、データのバックアップを実施します。
最後に、作業前の数に Pod 数を戻します。

危険

本手順では、サービスの停止が発生します。

2.4. バックアップ¶

2.4.1. レプリカ数の確認¶

作業前のレプリカ数の確認をし、状態を記録します。
リスト 2.1 コマンド¶
RS_AE=`kubectl get deploy ita-by-ansible-execute -o jsonpath='{@.spec.replicas}{"\n"}' -n exastro`
RS_ALRV=`kubectl get deploy ita-by-ansible-legacy-role-vars-listup -o jsonpath='{@.spec.replicas}{"\n"}' -n exastro`
RS_ATS=`kubectl get deploy ita-by-ansible-towermaster-sync -o jsonpath='{@.spec.replicas}{"\n"}' -n exastro`
RS_CS=`kubectl get deploy ita-by-conductor-synchronize -o jsonpath='{@.spec.replicas}{"\n"}' -n exastro`
RS_MC=`kubectl get deploy ita-by-menu-create -o jsonpath='{@.spec.replicas}{"\n"}' -n exastro`
RS_PA=`kubectl get deploy platform-auth -o jsonpath='{@.spec.replicas}{"\n"}' -n exastro`

2.4.2. リバースプロキシの停止¶

リバースプロキシ (platform-auth) のレプリカ数を 0 に変更し、エンドユーザーからのアクセスを制限します。
リスト 2.2 コマンド¶
kubectl scale deployment platform-auth -n exastro --replicas=0

2.4.3. バックヤード処理の停止¶

バックヤード処理 (ita-by-***) のレプリカ数を 0 に変更し、データベースの更新を停止します。
リスト 2.3 コマンド¶
1kubectl scale deployment ita-by-ansible-execute -n exastro --replicas=0
2kubectl scale deployment ita-by-ansible-legacy-role-vars-listup -n exastro --replicas=0
3kubectl scale deployment ita-by-ansible-towermaster-sync -n exastro --replicas=0
4kubectl scale deployment ita-by-conductor-synchronize -n exastro --replicas=0
5kubectl scale deployment ita-by-menu-create -n exastro --replicas=0

2.4.5. バックアップ¶

Exastro システム内で、Exastro Platform と Exastro IT Automation でデータベースを共有するか、分離するかによって手順が異なります。

  1. メンテナンス用コンテナを作成します。

Exastro Platform のデータベース(Exastro IT Automation と共有)を mysqldump を使ってバックアップします。
リスト 2.6 コマンド¶
 1cat <<_EOF_ | kubectl apply -f - -n exastro
 2apiVersion: v1
 3kind: Pod
 4metadata:
 5  name: platform-db-backup
 6  namespace: exastro
 7spec:
 8  containers:
 9  - command:
10    - sh
11    - -c
12    args:
13    - |
14      #!/bin/bash
15      sleep 3600
16    env:
17    - name: DB_DATABASE
18      valueFrom:
19        configMapKeyRef:
20          key: DB_DATABASE
21          name: platform-params-pf-database
22    - name: DB_HOST
23      valueFrom:
24        configMapKeyRef:
25          key: DB_HOST
26          name: platform-params-pf-database
27    - name: DB_PORT
28      valueFrom:
29        configMapKeyRef:
30          key: DB_PORT
31          name: platform-params-pf-database
32    - name: STORAGEPATH
33      valueFrom:
34        configMapKeyRef:
35          key: STORAGEPATH
36          name: ita-params-ita-global
37    - name: DB_ADMIN_PASSWORD
38      valueFrom:
39        secretKeyRef:
40          key: DB_ADMIN_PASSWORD
41          name: platform-secret-pf-database
42    - name: DB_ADMIN_USER
43      valueFrom:
44        secretKeyRef:
45          key: DB_ADMIN_USER
46          name: platform-secret-pf-database
47    image: mariadb:10.9
48    imagePullPolicy: IfNotPresent
49    name: platform-db-backup
50    resources: {}
51    securityContext:
52      allowPrivilegeEscalation: false
53      readOnlyRootFilesystem: false
54      runAsGroup: 1000
55      runAsNonRoot: true
56      runAsUser: 1000
57    volumeMounts:
58    - mountPath: /storage
59      name: volume-ita-backup-storage
60  volumes:
61  - name: volume-ita-backup-storage
62    persistentVolumeClaim:
63      claimName: pvc-ita-global
64  restartPolicy: Always
65  securityContext: {}
66  serviceAccount: default
67  serviceAccountName: default
68_EOF_

  1. データベースのバックアップを取得します。

リスト 2.7 コマンド¶
kubectl exec -it platform-db-backup -n exastro -- sh -c 'mysqldump -h ${DB_HOST} -P ${DB_PORT} -u ${DB_ADMIN_USER} -p${DB_ADMIN_PASSWORD} --all-databases --add-drop-table' | gzip > mysqldump_platform-db_`date +"%Y%m%d-%H%M%S"`.sql.gz

  1. ファイルのバックアップを取得します。

リスト 2.8 コマンド¶
kubectl exec -i platform-db-backup -n exastro -- sh -c 'tar zcvf - ${STORAGEPATH}' > exastro_storage_backup_`date +"%Y%m%d-%H%M%S"`.tar.gz

  1. 作業用に起動した Pod を削除します。

リスト 2.9 コマンド¶
kubectl delete pod platform-db-backup -n exastro
